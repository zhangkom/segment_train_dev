# 直播人体分割数据规范

## 1. 目录规范

```text
data/live_portrait/
  images/
    train/
    val/
  masks/
    train/
    val/
```

文件名必须一一对应，例如：

- `images/train/000001.jpg`
- `masks/train/000001.png`

## 2. 标签定义

- `0`：背景
- `1`：人体前景

mask 必须满足：

- 单通道 PNG
- 不允许 RGB 彩色 mask
- 不允许出现 `0/1` 之外的值

## 3. 标注边界

标注时按直播效果优先：

- 头发算人体
- 耳饰、眼镜默认算人体前景
- 帽子默认算人体前景
- 手持物体不算人体
- 镂空区扣空
- 背景中的屏幕人物不计入前景

## 4. 数据切分

建议：

- `train`: 80%
- `val`: 20%

切分时注意避免同一直播片段前后帧同时落入 `train` 和 `val`，否则验证集会虚高。

## 5. 难例清单

直播场景优先覆盖以下难例：

- 发丝边缘
- 手臂快速运动
- 主播衣服与背景颜色接近
- 暗光 + 补光灯反差
- 多人同框
- 站播时下半身细腿区域
- 坐姿遮挡桌面
- 透明杯子、麦克风支架遮挡

## 6. 数据验收

每次新增数据后执行：

```bash
python tools/check_dataset.py --root data/live_portrait
```

验收通过标准：

- 图片和 mask 数量一致
- 文件名完全对应
- mask 中只有 `0/1`
- 类别像素占比没有明显异常
